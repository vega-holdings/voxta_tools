# OpenAI Compatible Endpoint URL Format

> OpenAI Compatible endpoints in Voxta typically require the /v1 suffix appended to the URL.

## Answer

When configuring the **OpenAI Compatible** service in Voxta for local backends, ensure the endpoint URL includes the `/v1` suffix after the port. 

Example format: `http://localhost:[port]/v1`
