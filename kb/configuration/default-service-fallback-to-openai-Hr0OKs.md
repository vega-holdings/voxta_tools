# Default Service Fallback to OpenAI

> Voxta attempts to use OpenAI by default if no other service is configured.

## Answer

If no specific service is provided or configured, Voxta defaults to attempting to use OpenAI. To resolve this, you must either:
1. Configure a local backend (e.g., KoboldCpp).
2. Provide a valid OpenAI API key.
